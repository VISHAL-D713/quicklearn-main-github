Introduction

What is OOPS

Object-Oriented Programming (OOP) is a way of writing code where we create objects that combine data and actions together. It helps organize code better by grouping related things into one unit. OOP also makes programs easier to reuse, manage, and scale for bigger projects.

What are 4 fundamental pillars of OOPS?

The four fundamental pillars of OOP are:

1. Encapsulation – It is the process of bundling data (variables) and methods (functions) into a single unit called a class. It also hides the internal details and only exposes necessary parts, keeping data safe.

2. Abstraction – It focuses on showing only the essential features of an object while hiding the complex details. This makes the program easier to understand and reduces unnecessary complexity.

3. Inheritance – It allows one class to reuse the properties and methods of another class. This helps in reducing code duplication and builds a parent-child relationship.

4. Polymorphism – It means "many forms" and allows the same function or method to behave differently based on the object that calls it. This makes code more flexible and dynamic.

Real-life Analogies

1. Encapsulation – A bank account hides details like balance and only allows access through methods like deposit() or withdraw().

2. Abstraction – When you drive a car, you only use the steering wheel and pedals, but don't worry about the engine's inner working.

3. Inheritance – A dog is an Animal, so it inherits traits like eating and breathing, but also has its own unique features like barking.

4. Polymorphism – The word "draw" can mean drawing a shape on paper, or drawing money from an ATM, depending on the context.

Code Examples

Encapsulation


class Student:
    def __init__(self, name, age):
        # Private variable (encapsulation)
        self.__name = name
        self.__age = age
    
    def get_details(self):
        return f"Name: {self.__name}, Age: {self.__age}"

# Create object
s1 = Student("Rahul", 20)

# Access data through method (not directly)
print(s1.get_details())

Abstraction


from abc import ABC, abstractmethod

class Animal(ABC):
    @abstractmethod
    def sound(self):
        pass

class Dog(Animal):
    def sound(self):
        return "Bark"

class Cat(Animal):
    def sound(self):
        return "Meow"

# Example
animals = [Dog(), Cat()]
for a in animals:
    print(a.sound())

Polymorphism


class Shape:
    def area(self):
        pass

class Square(Shape):
    def __init__(self, side):
        self.side = side
    def area(self):
        return self.side * self.side

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    def area(self):
        return 3.14 * self.radius * self.radius

# Example
shapes = [Square(4), Circle(5)]
for s in shapes:
    print(s.area())

Inheritance


class Vehicle:
    def __init__(self, brand):
        self.brand = brand

    def show(self):
        return f"Brand: {self.brand}"

class Car(Vehicle):   # Inheriting Vehicle
    def __init__(self, brand, model):
        super().__init__(brand)
        self.model = model

    def show(self):
        return f"Car: {self.brand}, Model: {self.model}"

# Example
c = Car("Toyota", "Corolla")
print(c.show())
